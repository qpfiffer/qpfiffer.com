Notes for Gentoo in Production

gentoo_in_production.markdown by NO AUTHOR

The Plan

Use Wireguard to stick all services together.

Storage Server

  • Backups of VPS
  • Automated backups of laptop
  • MZBH
  • git hosting (git.sr.ht?)
  • Overlay hosting
  • MinIO file hosting

Build Server

  • Nightly builds of Node project
    • Should logpurge them unless they’re starred
    • Should have cron output attached to builds

VPS

  • Status page for all services
  • All external stuff

Real-world Nodes

  • Raspberry Pis with Hardened MUSL, minimal builds
  • Admin page
  • WiFi compromiser thing- Handshakes, automated connections, etc.

Steps

  1. Use an external binhost
  2. Use your own overlay for packaging your binaries
  3. Consider a hardened kernel with SELinux
  4. Look into the FEATURES="test" flag - any package failing it’s own unit tests will not be installed.
